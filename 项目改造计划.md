**5. 一套最适合你当前项目的本地化最终结构**

最终建议你走成这种形态：

- backend/：只做 API / auth / route / task status
- services/analysis/：新闻、群聊、信号、股票分析
- services/agent_service/：多角色、走势、研究问答
- services/reporting/：日报、研究报告、导出
- services/notifications/：企微/飞书/TG/邮件/实验微信
- services/execution/：仿真执行、风控、订单日志
- collectors/：所有数据抓取
- jobs/：任务注册与运行器
- skills/：策略技能、分析模板
- apps/web/：唯一前端工作台

这条线的好处是：

- 你的现有资产几乎都能保住
- 每次新增功能不再新增一个散脚本
- 以后无论接更多 LLM、更多数据源、还是接仿真交易，都不会把工程拖乱
