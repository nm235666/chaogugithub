# 脚本分层与 CLI 规范（v1）

更新时间：2026-03-31 UTC

## 1. 分层约定

- `fetch_*`：只负责抓取原始数据，不做重计算。
- `cleanup_*`：只负责清洗、去重、归档。
- `llm_*`：只负责评分、标签、摘要等模型推理。
- `build_*`：只负责聚合、信号构建、衍生表生产。
- `backfill_*`：只负责补历史缺口（可大批量）。
- `run_*`：只负责组合调用，不写复杂业务逻辑。
- `seed_*`：只负责一次性初始化。

## 2. CLI 统一参数（建议）

- `--limit`：控制处理上限。
- `--retry`：网络或外部接口失败重试次数。
- `--sleep`：请求节流间隔。
- `--dry-run`：只打印计划动作，不落库。

> 新增脚本建议复用根目录 `script_cli.py` 中的 `add_common_cli_flags` 与 `log_step`。

## 3. 退出码规范

- `0`：成功。
- `1`：业务失败（可重试/需人工处理）。
- `2`：参数错误（无需重试）。

## 4. 日志规范

- 每个阶段必须至少打印 `start` / `done` / `error`。
- 日志默认 UTC 时间，便于多机排障。
- 关键统计（新增/更新/失败）必须结构化输出。

## 5. 迁移建议（增量）

- 第一批：`fetch_*` 与 `llm_*` 新改脚本全部接入统一参数。
- 第二批：`backfill_*` 统一补 `--dry-run`。
- 第三批：`run_*` 统一输出子命令回放，便于追踪故障。
