# 距离终局目标剩余 10%~13% 的收敛计划（2026-04-17）

> 对齐文档：`docs/project_final_state_projection_2026-04-15.md`  
> 当前状态：核心 smoke 已达 `60/60`，整体完成度约 `87%~90%`。  
> 本文目标：把剩余差距收敛到“可验收、可回归、可提交”。

---

## 1. 仍未达到终局的关键点

1. 首页默认决策主线未完全统一（不同角色落点仍存在分叉）。
2. 决策可追踪率与闭环完成率缺少正式抽样统计证明（功能有，验收证据不足）。
3. 后台可观测能力未完全收敛为“单页可巡检”。
4. 回归门禁虽能跑通，但截图回归与高风险阻断机制仍偏弱。

---

## 2. 冲刺批次（建议 4 批）

## 批次 A：入口统一（目标回收 3%）

### 目标
- 让 admin/pro/limited 都有一致的“决策主线入口认知”，降低入口分叉带来的路径漂移。

### 实施范围
- `apps/web/src/app/navigation.ts`
- `apps/web/src/pages/dashboard/DashboardPage.vue`
- 必要时补 `apps/web/tests/e2e/smoke.spec.ts`

### 验收标准
1. admin/pro 登录后都可在首屏 1 次点击内进入 `research/decision`。
2. limited 继续保持权限边界，不出现越权入口。
3. `smoke` 中登录与默认落点用例稳定通过。

---

## 批次 B：闭环证据化（目标回收 3%~4%）

### 目标
- 把“看起来已闭环”升级为“可量化证明已闭环”。

### 实施范围
- `research/decision` 页面（仅补统计展示或导出辅助，不改后端协议）
- 增加审计脚本或测试：
  - 决策可追踪率抽样（20 条）
  - 动作闭环完成率抽样（20~30 条）

### 验收标准
1. 决策可追踪率（`action_id/run_id/snapshot_id + 证据`）>= 95%。
2. 闭环完成率（动作有后续状态与复盘结果）>= 85%。
3. 产出固定格式报告文件（日期版），可被后续回归复用。

---

## 批次 C：可观测单页巡检（目标回收 2%~3%）

### 目标
- 将 worker/queue/error concentration 收敛到单页主视图，避免多页跳转排障。

### 实施范围
- 优先在 `system` 增加聚合巡检卡区（可复用已有接口，不做后端重构）
- 候选页面：
  - `apps/web/src/pages/system/JobsOpsPage.vue`
  - `apps/web/src/pages/system/SourceMonitorPage.vue`

### 验收标准
1. 在单页可见：
  - worker alive / heartbeat
  - queue pending/running/error
  - recent error concentration
2. 出现异常时，页面给出明确下一步动作（去哪里修、先看什么）。

---

## 批次 D：回归门禁强化（目标回收 2%~3%）

### 目标
- 从“能跑”升级到“能稳态阻断高风险回归”。

### 实施范围
- `apps/web/tests/e2e/*`
- `apps/web/test-results/` 产物策略（失败留档、通过清理）
- 必要时补 `run` 脚本或 CI 命令约定

### 验收标准
1. `npm run build` 连续 3 次通过。
2. `npm run smoke:e2e:all` 连续 2 轮全绿（60/60）。
3. 图谱、决策板、用户管理写操作三类高风险页有截图回归留档。

---

## 3. 完成定义（Definition of Done）

满足以下条件后，可判定达到终局文档的 `95%+` 区间：

1. 入口：角色落点与决策主线一致，权限无回退。
2. 闭环：追踪率/闭环率达到阈值并有报告留档。
3. 可观测：关键健康项单页可巡检且有异常动作指引。
4. 回归：build + smoke 连续达标，关键截图回归可复验。

---

## 4. 风险与回滚点

1. 风险：入口统一可能影响已有角色习惯路径。  
- 回滚点：保留旧落点兼容开关，按角色渐进切换。

2. 风险：为达成抽样指标临时“修展示不修根因”。  
- 回滚点：报告中强制区分“展示修复”和“根因修复”。

3. 风险：回归门禁强化导致交付速度下降。  
- 回滚点：保留“快速 smoke”和“全量 smoke”双档策略。

---

## 5. 推荐执行顺序

`A 入口统一 -> D 回归门禁强化 -> B 闭环证据化 -> C 可观测单页巡检`

说明：
- 先稳入口与回归，避免后续改动引发回退；
- 再补闭环证据和可观测收敛，完成终局验收闭环。
