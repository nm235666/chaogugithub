# Zanbo Quant Web 全量重测报告（2026-04-06 UTC）

# 测试目标理解
本轮按“新的全量回归”执行，重新实测全站，不沿用上一轮通过结论。  
优先保障前端展示业务稳定（可见、可交互、信息表达一致），其次保障测试可用性（构建、smoke、权限矩阵、下载、异步链路可复验）。

- 登录 / 退出 / 首页 / 导航
- 股票列表 / 股票详情 / 价格中心 / 综合评分
- 国际资讯 / 国内资讯 / 个股新闻 / 日报总结

- 信号质量审计 / 配置 / 状态时间线

- 下载文件内容验证
- 接口级 smoke
- 前端构建验证
- 异步任务全链路验证

# 风险点
- 首页与个别页面存在加载态时间较长，短等待可能误判。
- 多角色异步任务终态耗时长，短观察窗口可能只能看到中间阶段。
- 高副作用写操作会影响真实数据，不适合本轮直接执行。

# 实施计划
- 批次 1：登录、首页、行情资讯主链路
- 批次 2：研究/信号/群聊主链路
- 批次 3：系统管理主链路
- 批次 4：权限矩阵、下载、smoke、构建、异步链路

# 网站模块清单
- 业务模块：行情、资讯、研究、信号、群聊
- 管理模块：监控、权限、任务、用户、邀请码、数据库审计、LLM 管理
- 验证模块：权限矩阵、下载、接口 smoke、前端构建、异步任务

# 核心业务流程
登录 -> 导航进入各业务页 -> 列表/详情/筛选 -> 研究与信号分析 -> 系统管理读路径 -> 非管理员路由访问验证 -> 下载导出 -> 异步任务提交与轮询。

# 高风险功能点
- 路由权限边界（`/dashboard`、`/system/*`）
- 角色权限策略展示与保存口径
- 异步任务阶段推进与终态收敛
- 高副作用写操作（保存/删除/触发真实任务）
