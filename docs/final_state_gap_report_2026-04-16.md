# 距离终局目标的差距清单（2026-04-16）

> 依据文档：`docs/project_final_state_projection_2026-04-15.md`
> 目标：只写“差的部分/未达标部分”，用于下一轮整改。

## 1) 产品形态差距（当前约 65%，缺口 35%）

### 1.1 首页仍未形成“今日决策面板”单入口
- 现状：核心入口仍分散在 stocks/signals/research/intelligence 多页面。
- 影响：用户需要自己拼接流程，决策效率和一致性下降。
- 证据：路由与导航仍为模块化分入口（`apps/web/src/app/router.ts`、`apps/web/src/app/navigation.config.json`）。
- 优先级：P1
- 补齐标准：登录后默认进入决策工作台，首屏展示“今日候选/动作/风险/待办”。

### 1.2 任务上下文跨页传递不完整
- 现状：虽有部分上下文透传，但并非所有入口都带“来源任务+当前目标+下一步动作”。
- 影响：进入决策页后常回到“通用态”，上下文连续性不足。
- 证据：不同入口存在不同 query 透传策略，未统一契约。
- 优先级：P1
- 补齐标准：所有主入口统一 `DecisionContext` 契约并可一键还原/清空。

---

## 2) 业务闭环差距（当前约 55%，缺口 45%）

### 2.1 “跟踪 → 反馈 → 复盘”链路稳定性不足
- 现状：动作可记录，但后续验证、结果追踪与复盘沉淀不稳定。
- 影响：系统容易停留在“有结论”，难形成“结论是否有效”的组织记忆。
- 证据：决策校准与验证链路虽有新增，但仍有运行态不稳定与数据稀疏场景。
- 优先级：P0
- 补齐标准：每条关键动作默认可追踪到状态、收益、命中、复盘结论。

### 2.2 因子链路闭环断裂（worker 离线 + pending 积压）
- 现状：`quantaalpha` worker 非常驻，`hybrid` 模式下任务入队后可能无人消费。
- 影响：出现“可发起任务但不执行”的伪闭环。
- 证据：`WORKER_OFFLINE_BACKLOG` 告警、`pending` 存量、心跳长期过期（`services/quantaalpha_service/service.py`）。
- 优先级：P0
- 补齐标准：worker 常驻守护 + 心跳健康 + pending 自动消化 + 失败可重试可解释。

### 2.3 错误语义仍有“UNKNOWN_ERROR”聚集
- 现状：错误码聚集在 `UNKNOWN_ERROR`，异常归因粒度不足。
- 影响：排障效率低，复盘难定位真实根因。
- 证据：健康接口错误码集中告警（`ERROR_CODE_CONCENTRATED`）。
- 优先级：P1
- 补齐标准：将 unknown 拆分为可行动错误族（配置/数据/超时/外部依赖/中断恢复）。

## 3) 模块打通差距（当前约 58%，缺口 42%）

### 3.1 news → decision 仍偏“跳转”，缺少结构化动作沉淀
- 现状：已可跳决策页，但新闻结论未标准化为“动作对象”。
- 影响：信息消费强于行动闭环，难积累可复用资产。
- 证据：新闻页存在入口按钮，但动作模板/责任/时点/验证目标不统一。
- 优先级：P1
- 补齐标准：新闻触发动作模板化（动作类型、理由、时限、验证指标）。

### 3.2 chatrooms → decision 仍偏“讨论导流”，缺少结论结构化
- 现状：有“去决策板”入口，但会话结论未标准化沉淀。
- 影响：讨论结果难追踪、难统计、难复盘。
- 证据：聊天室链路仍以浏览与跳转为主，结构化输出弱。
- 优先级：P1
- 补齐标准：会话可一键生成“结论卡 + 动作卡 + 风险卡”并入决策流。

### 3.3 signals 与 decision 的联动仍缺“统一动作协议”
- 现状：图谱可跳决策板，但关系强度、优先级未直接映射为动作建议。
- 影响：从关系发现到执行动作仍需手动判断。
- 证据：图谱链路强化了展示与筛选，但动作化规则未统一。
- 优先级：P2
- 补齐标准：信号节点支持标准动作建议（观察/复核/确认/拒绝）并携带依据。

---

## 4) 架构与可观测差距（当前约 52%，缺口 48%）

### 4.1 关键后台任务缺“默认守护”
- 现状：部分 worker 已有 guard（如 multi-role、roundtable），但 quant worker 未纳入同等级守护。
- 影响：运行中断后无自愈，形成业务盲区。
- 证据：`job_registry.py` 中 quanta 任务存在但未形成 worker 守护闭环。
- 优先级：P0
- 补齐标准：关键 worker 全部纳入守护矩阵，具备异常拉起与告警联动。

### 4.2 写操作观测不统一
- 现状：部分页面已补“pending/success/error”，但仍非全站统一协议。
- 影响：用户对“是否生效”存在不确定性。
- 证据：system 页面改造进度不一致。
- 优先级：P1
- 补齐标准：统一 `ActionStatus` + 最近生效时间 + 变更摘要回显。

### 4.3 指标存在“统计口径碎片化”风险
- 现状：局部页面已加口径说明，但跨模块统计规则未完全统一。
- 影响：同一业务问题在不同页结论不一致，信任感受损。
- 证据：图谱/列表/决策统计仍有页面级实现差异。
- 优先级：P1
- 补齐标准：统一指标字典与口径来源，页面仅消费统一口径。

## 5) 验收与回归差距（当前约 50%，缺口 50%）

### 5.1 构建门禁未稳定通过
- 现状：`npm run build` 当前存在 TS 错误（未使用变量）。
- 影响：发布稳定性与回归可信度下降。
- 证据：本地构建输出 `TS6133`。
- 优先级：P0
- 补齐标准：构建全绿，禁止带编译错误合入。

### 5.2 smoke 回归非全绿
- 现状：`smoke:e2e:all` 为 57 过 / 3 失败。
- 影响：主链路行为仍有回归风险（登录默认落点、写操作边界）。
- 证据：失败用例包括：
  - `admin 登录后落到 dashboard`
  - `pro 默认落点正确且可打开评分总览`
  - `重复提交时按钮会进入进行中状态`
- 优先级：P0
- 补齐标准：smoke 全通过并纳入 CI 阻断。

### 5.3 “审美评审 + 截图回归”尚未制度化为强门禁
- 现状：有人工评审实践，但未形成每轮强制门禁与评分归档标准。
- 影响：视觉一致性和产品感易回退。
- 证据：存在专项评审产出，但流程化程度不足。
- 优先级：P2
- 补齐标准：关键路由截图回归 + 8 维评分达阈值后方可收敛。

---

## 6) 按影响排序的优先整改（只列差距项）

### P0（先修，不修会持续掉分）
1. quantaalpha worker 常驻/守护缺失，导致 pending 堆积与闭环断裂。
2. build 不通过（TS 错误）与 smoke 非全绿（3 个失败）。
3. 决策跟踪-反馈-复盘链路稳定性不足（动作后证据链不稳）。

### P1（修完可显著提升“决策操作系统”质感）
1. news/chatrooms 到 decision 的结构化动作沉淀。
2. 全站写操作状态机与生效回显统一。
3. 指标口径统一化与跨页一致性治理。
4. 首页“决策中台化”入口收敛。

### P2（用于拉高天花板）
1. signals 到动作建议的自动化策略映射。
2. 审美评分与截图回归门禁化、持续化。

### P3（完成所有功能的闭环）
1. 使系统整体从“可用的半闭环投研系统”升级为“稳定可运行的全闭环决策操作系统”。
2. 全结论对象化、全动作闭环化、全结果可复盘化。
3. 全站状态协议、错误语义、指标口径、观测门禁统一。
