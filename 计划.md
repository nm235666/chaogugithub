把 TradingAgents 的“多代理决策能力” + daily_stock_analysis 的“工程化配置/报告/回测能力”，融合到你当前 Zanbo Quant 里，形成“可解释、可回测、可运维”的投研闭环。

Phase 1（1-2周）先做高收益低风险

统一 LLM 配置与调用策略
引入“主模型 + fallback + 渠道”统一配置层（借鉴 daily_stock_analysis 的 channels 思路）。
输出字段统一包含 used_model、attempts、latency_ms。
目标：所有 llm_*.py 与后端 LLM API 同一套路由策略。
LLM 用量与成本审计
新增 llm_usage（任务名、页面、模型、tokens、耗时、成功率、错误码）。
新增 /api/llm/usage/summary 和前端审计页。
目标：看得见成本瓶颈，支持限流/调度优化。
报告完整性校验引擎
在日报总结、多角色分析、走势分析后加“结构校验 + 缺失重试 + 占位兜底”。
目标：减少“文档不全、前端渲染失败、下载内容缺段”。
Phase 2（2-4周）接入 TradingAgents 核心能力

多角色编排器升级为“专职代理流水线”
角色建议：宏观、行业、技术、资金、新闻、风险、组合。
每个角色输出结构化 JSON，最后由“决策代理”汇总。
目标：从“单段结论”变成“可追踪角色贡献”。
Bull/Bear 对辩机制
对同一标的生成多头/空头两组证据，再由裁判输出最终方向与置信度。
落库到 investment_signal_events 的“证据链字段”。
目标：提升信号可解释性和鲁棒性。
风控闸门（Risk Gate）
在最终信号前加入硬规则/软规则（波动、流动性、事件风险、消息可信度）。
目标：过滤高噪声信号，避免“看起来强但不可执行”。
Phase 3（3-5周）形成“信号-结果”闭环

信号状态机生产化
状态：初始→强化→弱化→证伪→反转。
每日基于新增证据自动迁移状态。
目标：从静态榜单升级为动态演化追踪。
标准投研报告生成器
输入：数据摘要 + 事件链 + 多角色观点 + 风险提示 + 相关标的。
输出：统一 Markdown/图片/PDF（下载与推送一致）。
目标：所有页面报告模板统一，减少维护成本。
回测与归因
对“信号方向/置信度/持有窗口”做命中率和收益归因。
目标：让模型与规则可持续优化，而不是只看当下结果。
Phase 4（持续）高级能力

主题引擎 + 市场预期层
主题聚合（黄金/军工/AI/航运等）+ 外部预期数据（后续可接 Polymarket 类）。
目标：事件发生 vs 市场押注 同屏观察。
记忆与自校准
记录“哪些来源/角色/提示词组合最有效”，动态调权。
目标：长期提升分析稳定性。
运维化完善
Source 健康分、任务 SLA、积压告警、自动降级。
目标：系统高可用与可观测。
落地顺序（你现在最适合）

LLM配置统一 + 用量审计
报告完整性校验
多代理流水线 + 对辩 + 风控闸门
状态机 + 回测归因
