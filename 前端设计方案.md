### **最终架构方案：Zanbo Quant NextGen**

该方案的核心是采用 **Vue 3** 生态系统，并结合 **TypeScript** 和 **Tailwind CSS**，构建一个数据驱动的、组件化的单页面应用（SPA）。

#### **一、 核心技术选型 (Core Technology Stack)**

| 类别 (Category) | 技术选型 (Technology) | 解决的问题与优势 (Benefits & Rationale) |
| :--- | :--- | :--- |
| **前端框架** | **Vue 3 (Composition API)** | 替代原生JS，提供声明式、组件化的开发模式。Composition API 能更好地组织复杂逻辑，代码可维护性极强。 |
| **开发语言** | **TypeScript** | 为 JavaScript 提供静态类型检查，在编译阶段就能发现大量错误，极大提升代码健壮性和团队协作效率。 |
| **构建工具** | **Vite** | 提供闪电般的开发服务器启动速度和极速的热模块更新（HMR），显著提升开发体验。 |
| **UI 框架/库** | **Tailwind CSS + Headless UI** | 解决UI不统一问题。Tailwind 提供原子化 CSS 类，实现快速、高度一致的 UI 构建。Headless UI 提供无样式的、功能完备的组件（如下拉菜单、模态框），与 Tailwind 完美集成。 |
| **路由管理** | **Vue Router** | 实现单页面应用。页面切换无需整页刷新，体验如原生应用般流畅，并提供强大的路由守卫、参数传递等功能。 |
| **状态管理** | **Pinia** | 替代散乱的全局变量，提供中心化的、类型安全的状态管理。逻辑清晰，易于调试，方便在不同组件间共享状态（如用户信息、系统健康度）。 |
| **HTTP 客户端** | **Axios** | 替代原生 `fetch`。提供更强大的 API，如请求/响应拦截器、超时设置、取消请求、统一的错误处理等。 |
| **图表库** | **保留并封装 (Wrapper)** | 保留现有的 `Lightweight Charts`, `Chart.js`, `Marked.js` 等，但将它们封装成独立的 Vue 组件，隔离其命令式API，使其能以声明式的方式在 Vue 应用中使用。 |

#### **二、 应用架构设计 (Application Architecture)**

##### **2. 数据流与状态管理**

数据流将遵循单向数据流原则，清晰可控：

1.  **用户交互**: 在 `View` 层组件中触发一个事件（如点击“分析”按钮）。
2.  **调用服务**: 事件处理器调用 `services/api.ts` 中对应的 API 请求函数。
3.  **API 请求**: `api.ts` 使用 Axios 发送 HTTP 请求到后端。
4.  **更新状态**: 请求成功后，调用 `store/system.ts` 中的一个 `action`，将返回的数据更新到 Pinia 的 `state` 中。
5.  **响应式更新**: 所有订阅了该 `state` 的 `View` 或 `Component` 都会自动、高效地更新其视图，展示新数据。

对于 WebSocket，`services/websocket.ts` 将负责维护连接，并将收到的消息分发给相关的 Pinia store 去更新状态。

##### **3. 组件化策略**

*   **布局组件 (`layouts/`)**: 负责应用的整体结构，如 `DefaultLayout.vue` 包含一个固定的侧边栏、一个顶部栏和一个用于显示当前页面的 `<router-view>` 插槽。
*   **页面/视图组件 (`views/`)**: 对应每个功能模块，是“智能组件”。它们负责从 Pinia store 获取数据、调用 services，并将数据传递给基础组件。
*   **基础组件 (`components/`)**: 是“哑组件”。它们只负责展示数据（通过 props 接收）和派发事件（通过 emit）。例如，`Button.vue` 只关心自己的外观和点击事件，不关心点击后具体做什么。

#### **三、 关键模块实现思路**

##### **1. 统一 API 服务层 (`services/api.ts`)**

```typescript
// services/api.ts
import axios from 'axios';
import { useUiStore } from '@/store/ui';

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL, // 从环境变量读取
  timeout: 10000,
});

// 请求拦截器：显示全局加载状态
apiClient.interceptors.request.use(config => {
  useUiStore().setLoading(true);
  return config;
});

// 响应拦截器：处理错误，关闭加载状态
apiClient.interceptors.response.use(
  response => {
    useUiStore().setLoading(false);
    return response.data;
  },
  error => {
    useUiStore().setLoading(false);
    // 在此统一处理错误，例如弹出 Toast 通知
    console.error('API Error:', error);
    return Promise.reject(error);
  }
);

// 导出具体的 API 函数
export const getDashboardData = () => apiClient.get('/api/dashboard');
export const startMultiRoleAnalysis = (params) => apiClient.get('/api/llm/multi-role/start', { params });
```

##### **2. 封装图表为 Vue 组件 (`components/charts/KLineChart.vue`)**

```vue
<template>
  <div ref="chartContainer" class="w-full h-full"></div>
</template>

<script setup lang="ts">
import { ref, onMounted, onUnmounted, watch } from 'vue';
import { createChart, IChartApi, ISeriesApi } from 'lightweight-charts';

// 定义 props 接收K线数据
const props = defineProps<{
  klineData: any[]; // 你的K线数据类型
  volumeData: any[]; // 成交量数据类型
}>();

const chartContainer = ref<HTMLElement | null>(null);
let chart: IChartApi | null = null;
let candleSeries: ISeriesApi<'Candlestick'> | null = null;

onMounted(() => {
  if (chartContainer.value) {
    chart = createChart(chartContainer.value, { /* ...chart-options... */ });
    candleSeries = chart.addCandlestickSeries({ /* ...series-options... */ });
  }
});

// 监听数据变化，并更新图表
watch(() => props.klineData, (newData) => {
  if (candleSeries) {
    candleSeries.setData(newData);
  }
}, { deep: true });

onUnmounted(() => {
  if (chart) {
    chart.remove();
    chart = null;
  }
});
</script>
```

##### **3. 实时通信组合式函数 (`composables/useWebSocket.ts`)**

```typescript
// composables/useWebSocket.ts
import { ref, onMounted, onUnmounted } from 'vue';
import { useSystemStore } from '@/store/system';

export function useWebSocket() {
  const isConnected = ref(false);
  let socket: WebSocket | null = null;

  const connect = () => {
    socket = new WebSocket('ws://your-server/ws/realtime');
    
    socket.onopen = () => { isConnected.value = true; };
    socket.onclose = () => { isConnected.value = false; };

    socket.onmessage = (event) => {
        const msg = JSON.parse(event.data);
        // 根据消息类型，调用 Pinia store 的 action
        if (msg.event === 'job_run_update') {
            useSystemStore().fetchDashboard(); // 例如，重新拉取仪表盘数据
        }
    };
  };

  onMounted(connect);
  onUnmounted(() => socket?.close());

  return { isConnected };
}
```
